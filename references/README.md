# References

Cross-skill reference material lives here. Single-skill reference material lives next to the skill that owns it (e.g., `skills/<name>/<topic>.md`).

A file belongs in this directory when two or more skills point at it. Until that happens, keep it co-located with the skill.

## Files

- `testing-patterns.md` — Test pyramid, sizes, AAA structure, naming, anti-patterns. Used by `test-driven-development`, `requesting-code-review`, and the `test-engineer` persona.
- `security-checklist.md` — Pointer to the canonical security checklist (lands with the `security` skill).
- `performance-checklist.md` — Pointer to the canonical performance checklist (lands with the `performance` skill).
- `accessibility-checklist.md` — Pointer to the canonical accessibility checklist (lands with the `frontend-ui` skill).
